# Kimi Subagents — Track A Overlay

[![CI](https://github.com/null0xxx/Kimi_subagents/actions/workflows/check.yml/badge.svg)](https://github.com/null0xxx/Kimi_subagents/actions/workflows/check.yml)

This project is a **Kimi Code CLI overlay** that keeps root context healthy by delegating work through a strict `explore` → `plan` → `coder` subagent chain. The root agent handles orchestration only; all research, design, and implementation are pushed to read-only or file-editing subagents, with critical state externalized into `SetTodoList` and file artifacts.

## What is Track A — No-Patch Overlay?

Track A is the approved build path: it uses only existing Kimi extension points (`Agent`, `SetTodoList`, background tasks, plan mode, and hierarchical `AGENTS.md` injection) without modifying the Kimi Code CLI bundle, Python code, or YAML specs. It adds conventions and documentation rather than source patches or custom agent runtimes.

## Key files

- `AGENTS.md` — Root orchestration spec: roles, task packet format, output contract, TODO/artifact rules, parallelism policy, and failure handling.
- `scripts/check-artifact-naming.py` — Validates artifact filenames in `analysis/` and `design/` against `design/artifact-conventions.md`.
- `Makefile` — Thin wrappers: `make check-artifacts` and `make check` run the naming checker; `make check-strict` runs it in strict mode; `make test` runs the unit tests; `make check-shell` validates shell script syntax; `make ci` runs the full local CI pipeline; `make clean` removes Python cache artifacts.
- `analysis/` — Research, validation reports, test results, background summaries, and execution reports.
- `design/` — Architecture blueprints, implementation plans, conventions, and session state.

## Running the naming checker

Directly with Python:

```bash
python3 scripts/check-artifact-naming.py
```

Via Make:

```bash
make check-artifacts
make check          # alias for make check-artifacts
```

Run `make help` to list all available targets and their descriptions.

Strict mode (warnings become fatal, useful for CI or pre-commit hooks):

```bash
python3 scripts/check-artifact-naming.py --strict
make check-strict
```

## Running the tests

The unit tests for the naming checker live in `scripts/test-check-artifact-naming.py`:

```bash
make test
# or directly
python3 scripts/test-check-artifact-naming.py
```

## Continuous Integration

A GitHub Actions workflow runs `make ci` (`make check-strict`, `make test`, and `make check-shell`) on every push and pull request to `master`. See `.github/workflows/check.yml` for details.

To reproduce the CI pipeline locally, run:

```bash
make ci
```

## Optional pre-commit hook

You can opt in to run `make ci` automatically before each commit:

```bash
./scripts/install-hooks.sh
# or
make install-hooks
```

This configures Git to use the hooks in `.githooks/`. To disable later:

```bash
git config --unset core.hooksPath
```

This runs `make ci` (which runs `make check-strict`, `make test`, and `make check-shell`); if it fails, the commit is rejected and the output shows the failure.

A successful run prints the normal output of all three checks, ending with the unit-test summary:

```
Ran 24 tests in ...s
OK
```

## Cleaning up

To remove Python cache artifacts (`__pycache__` directories and `*.pyc` / `*.pyo` files) generated by the test suite, run:

```bash
make clean
```

## What this project does NOT do

- **No source patching** — Do not modify Kimi Code CLI bundles, Python code, or YAML specs.
- **No custom agents** — Only the verified `explore`, `plan`, and `coder` subagent types are used.
- **No nested subagents** — Subagents do not create their own subagents.
- **No unsupported tool injection** — Do not add custom tools, force-enable `Think`, or rely on unverified memory channels.
